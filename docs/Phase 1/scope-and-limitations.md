# 2. Scope & Limitations

## 2.1 Major Features
- **FE‑1:** Family Member Management  
- **FE‑2:** Health Event Recording  
- **FE‑3:** Automatic Risk Detection  
- **FE‑4:** Personalized Recommendations  
- **FE‑5:** Awareness Hub  
- **FE‑6:** Clinic Appointment Simulation  
- **FE‑7:** Alert History  
- **FE‑8:** Role‑Based Access Control  
- **FE‑9:** Bilingual Interface  
- **FE‑10:** Mobile‑First Design  

---

## 2.2 Scope of Initial Release
The first release focuses on core preventive‑health functionality:

- Family member & health event management  
- Rule‑based risk detection  
- Personalized alerts  
- Awareness Hub  
- Basic clinic booking simulation  
- Bilingual interface  
- Authentication & RBAC  
- Mobile‑first UI  
- Core security measures  
- Usability foundations  

These features directly support early detection and user engagement.

---

## 2.3 Scope of Subsequent Releases
Future enhancements may include:

- Real clinic system integration  
- AI‑enhanced risk prediction  
- Push notifications  
- Multi‑family linking  
- Provider dashboards  
- National health system integration  
- Genetic‑risk education modules  
- Real‑time appointment availability  
- Secure document uploads  

These depend on regulatory approval and technical partnerships.

---

## 2.4 Limitations & Exclusions
To maintain academic feasibility, the following are excluded:

- No real medical diagnosis  
- No integration with hospital EHR systems  
- No AI/ML risk prediction  
- No offline mode  
- No real‑time clinic availability  
- Simulated notifications only  
- No genetic testing or lab data ingestion  
- No emergency‑response features  

These limitations keep the project focused while demonstrating meaningful value.
