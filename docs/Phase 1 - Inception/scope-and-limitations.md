# 2. Scope & Limitations

## 2.1 Major Features

The following features define the core functionality of the Sillah (صلة) platform:

- **FE‑1:** Family Member Management  
  Add and manage family members with demographic and health-related data.

- **FE‑2:** Health Event Recording  
  Record hereditary conditions such as heart disease, diabetes, hypertension, and high cholesterol.

- **FE‑3:** Automatic Risk Detection  
  Use rule-based logic to detect inherited health patterns and generate alerts.

- **FE‑4:** Personalized Recommendations  
  Provide tailored advice on screening, lifestyle changes, and follow-up actions.

- **FE‑5:** Awareness Hub  
  Offer educational content, checklists, and preventive health resources.

- **FE‑6:** Clinic Appointment Simulation  
  Allow users to simulate booking appointments with healthcare providers.

- **FE‑7:** Alert History  
  Display previously generated alerts and notifications for review.

- **FE‑8:** Role-Based Access Control (RBAC)  
  Assign different permissions to citizens, providers, and administrators.

- **FE‑9:** Bilingual Interface  
  Support both Arabic (RTL) and English (LTR) for full accessibility.

- **FE‑10:** Mobile-First Design  
  Prioritize mobile usability for widespread access across Saudi households.

---

## 2.2 Scope of Initial Release

The initial release focuses on delivering essential preventive-health functionality:

- **FE‑1:** Family member and health event management  
- **FE‑2:** Rule-based risk detection for cardiac disease, diabetes, hypertension, and high cholesterol  
- **FE‑3:** Personalized alerts and recommendations  
- **FE‑4:** Awareness Hub with curated educational content  
- **FE‑5:** Basic clinic booking simulation  
- **FE‑6:** Bilingual interface with correct RTL/LTR behavior  
- **FE‑7:** Authentication and role-based access control  
- **FE‑8:** Mobile-first responsive UI  
- **FE‑9:** Core security measures (input validation, password rules, session handling)  
- **FE‑10:** Usability foundations (clear error messages, intuitive navigation, visual feedback)

These features directly support early detection, user engagement, and prototype feasibility.

---

## 2.3 Scope of Subsequent Releases

Future releases may expand Sillah’s capabilities to support broader public-health integration and advanced analytics:

- **FE‑11:** Integration with real clinic systems and appointment APIs  
- **FE‑12:** AI-enhanced risk prediction models  
- **FE‑13:** Push notifications via SMS, WhatsApp, or national health platforms  
- **FE‑14:** Multi-family linking for extended households  
- **FE‑15:** Advanced dashboards for healthcare providers  
- **FE‑16:** National health system integration (e.g., Sehhaty, Mawid)  
- **FE‑17:** Genetic-risk education modules  
- **FE‑18:** Real-time appointment availability  
- **FE‑19:** Secure document uploads (e.g., lab results)

These enhancements depend on regulatory approval, technical partnerships, and infrastructure readiness.

---

## 2.4 Limitations & Exclusions

To maintain academic feasibility, the following items are excluded from the current version:

- **LI‑1:** No real medical diagnosis — alerts are screening suggestions only  
- **LI‑2:** No integration with hospital electronic health record (EHR) systems  
- **LI‑3:** No AI/ML-based risk prediction in the current version  
- **LI‑4:** No offline mode or local-device storage  
- **LI‑5:** No real-time clinic availability or live appointment confirmation  
- **LI‑6:** Notification delivery is simulated and not production-grade  
- **LI‑7:** No genetic testing or ingestion of laboratory data  
- **LI‑8:** No emergency-response features

These exclusions ensure the project remains focused, achievable, and aligned with SE311 academic constraints.
