# 3. Stakeholder Analysis

## 3.1 Stakeholder Identification

The following stakeholders were identified for the Sillah system:

- Families (Primary Users)
- Healthcare Providers
- System Administrators
- Legal/Regulatory Authorities (PDPL)
- Public Health Authorities (Future Stakeholders)

Only family members and healthcare providers participated directly in interviews.

---

## 3.2 Stakeholder Mapping Table

| Stakeholder | Role | Interest | Influence | Engagement Method |
|------------|------|----------|-----------|-------------------|
| Families (Primary Users) | Use the system to track hereditary health and receive alerts | Early detection, usability, privacy protection | High | Interview + Survey |
| Healthcare Providers | Validate medical screening logic and recommendations | Clinical accuracy | High | Interview |
| System Administrators | Manage user roles and system integrity | Secure access control | Medium | Document Analysis |
| Legal/Regulatory Authority (PDPL) | Ensure compliance with data protection laws | Protection of sensitive health data | High | Document Analysis |
| Public Health Authority | Future integration potential | Preventive health scaling | Medium | Document Analysis |
